# 📌 Devstagram

개발자들의 소통과 문화 향유를 지향하는 소셜 네트워크 서비스입니다.

---

## 📝 프로젝트 소개

- 인스타그램을 벤치마킹한 개발자 친화적인 소셜 네트워크 서비스 **Devstagram**이다.
- 인스타그램의 여러 기본적인 기능을 유사하게 구현하되, 개발 지식 공유와 코드 리뷰 기반의 소통을 지향하며, 여러 개발 문화를 접목하였다.

---

## 🚀 주요 기능

### ✅ 사용자 기능

- **계정 관리:** OAuth2 또는 JWT 기반의 회원가입 및 로그인, 개발자 프로필 설정.
- **피드 & 포스팅:** 코드 스니펫 지원 게시글 작성, 멀티미디어 업로드.
- **리액션 & 소통:** 게시글에 대해 좋아요, 댓글과 대댓글 작성.
- **네트워킹:** 실시간 DM, 게시글 및 스토리 공유를 통한 개발자 간 소통.

### ✅ 시스템 기능

- **좋아요 최적화:** Redis의 Set 자료구조와 Write-Behind 패턴을 활용한 중복 체크 및 DB 부하 분산.
- **피드 알고리즘**: 사용자의 게시글 조회 히스토리를 기반으로한 자체 알고리즘을 통한 피드.
- **코드 품질 자동화:** **Spotless**와 **Checkstyle**을 CI 파이프라인에 통합하여 코드 포맷팅 및 컨벤션 검사 자동 수행.
- **품질 관리 강제화:** 커밋 및 빌드 단계에서 품질 검사를 자동화하여 협업 시 일관된 코드 스타일과 안정성 유지.

### ✅ 인프라 기능

- **CI/CD 파이프라인:** **GitHub Actions**를 활용한 빌드, 테스트, 배포 자동화.
- **클라우드 아키텍처:** **AWS EC2**, **RDS**, **S3**를 활용한 확장 가능하고 안정적인 인프라 구성.
- **컨테이너 관리:** **Docker**를 통한 환경 격리 및 배포 이미지 경량화.

## 🏷️ Commit Convention

| Tag | Content |
| --- | --- |
| Feat | 기능 개발 |
| Fix | 코드 수정 |
| Bug | 예상하지 못한 문제 수정 |
| HotFix | 긴급 수정 |
| Style | 코드 스타일 변경 (로직 변경 없음) |
| Refactor | 기능 변화 없이 구조 개선 |
| Test | 테스트 코드 추가 / 수정 |
| CI/CD | 배포 / 빌드 / 파이프라인 변경 |
| Docs | 문서 작업 |

## 🌿 Git Flow Strategy

- main : 운영 브랜치
- dev : 개발 통합 브랜치
- feat/* : 기능 개발
- fix/* : 버그 수정
- hotfix/* : 긴급 수정
- refactor/* : 리팩토링
- test/* : 테스트 작업
